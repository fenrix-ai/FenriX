# Chef Roster — Random Generation Framework

**Game:** Bakery Bash
**Date:** April 15, 2026
**Purpose:** Reference table for the chef random generator — names, asset design descriptions, specialties, and skill multipliers by nationality and gender.

> Specialty products per nationality are **hidden from players** and should never be surfaced in the UI. Only nationality and skill level are shown.

---

## Chef Roster Reference Grid

> **Asset Design Note:** Each chef entry includes a primary archetype and one or more diverse variant suggestions. For culturally heterogeneous nationalities (French, American), multiple skin tone and hair type representations are listed — the generator should be able to spawn any variant. All descriptions are intended as direct input for front-end character design.
>
> **Scope:** Design the chef as a **head and neckline only** — no full body required. The crop should extend just past the collar so that neck attire (neckerchief, coat collar, neckband) is visible. Focus on facial features, hair, hat, and neckline detail. Think portrait-style avatar icon.

> **How to read the multipliers:** Base = output on non-specialty products. Spec = output on specialty products. An Intermediate French chef on Croissants outputs 1.75×; the same chef on Bagels outputs 1.25×. Players never see these numbers — they infer from throughput and satisfaction signals over time.

## Skill Multipliers

| Skill Level | Non-Specialty Output | Specialty Output |
|---|---|---|
| **Novel** | 1.0× | 1.4× |
| **Intermediate** | 1.25× | 1.75× |
| **Advanced** | 1.6× | 2.2× |

## Random Generation Logic (Implementation Reference)

```
1. Pick nationality at random (French / Japanese / Italian / American)
2. Pick gender at random (Male / Female)
3. Pick variant at random from the available variants for that nationality × gender
4. Pick name at random from the 2 options for that gender × nationality
5. Pick skill level weighted by current round spawn rates (Novel / Intermediate / Advanced)
6. Assign specialty products from the nationality's hidden specialty map
7. Apply multipliers from the skill level table above
8. Surface to UI: nationality + skill level only — never specialty or multiplier values
```

**Spawn rates by round:**

| Skill Level | Round 1 | Round 2 | Round 3 | Round 4 | Round 5 |
|---|---|---|---|---|---|
| **Novel** | 65% | 55% | 40% | 20% | 5% |
| **Intermediate** | 30% | 35% | 40% | 45% | 45% |
| **Advanced** | 5% | 10% | 20% | 35% | 50% |
